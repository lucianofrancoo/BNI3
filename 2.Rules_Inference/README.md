# 2. Boolean Rules Inference (`2.Rules_Inference/`)

This module is responsible for inferring the logical relationships (Boolean rules) between different genes/nodes over time based on the binarized data from the previous stage. The algorithm leverages **Gene Expression Programming (GEP)**, capable of discovering robust and complex underlying regulatory interactions matching the expression dynamics.

The pipeline in this step consists of four consecutive scripts: inference, analysis, evaluation, and visualization.

## Pipeline Scripts and Usage

### 1. Rule Inference (`1.BNI3_Boolean_Rules_Inference.py`)
This is the core algorithm. It runs the GEP evolution process to infer the Boolean transition rules using the original raw data and the binarized dataset. It supports optional Neural Network (MLP) validation and regulator connectivity penalization (favoring biological parsimony around $K \approx 2$).

**Basic Usage:**
```bash
python3 1.BNI3_Boolean_Rules_Inference.py -i ../1.Binarization/Example/Counts_lite.tsv -i_binary ../1.Binarization/Example/bin_SSD.tsv -o Example/
```
*Note: This script will run multiple independent repetitions per gene and automatically trigger the analysis module at the end of its execution.*

### 2. Results Analysis (`2.BNI3_Analyze_results.py`)
This script parses the multiple output iterations generated by the inference step. It looks at the resulting `.tsv` metrics and extracts the best Boolean rules based on maximum correct transitions (`n_correct`), regulator penalizations, and MLP validation scores.

**Basic Usage:**
```bash
python3 2.BNI3_Analyze_results.py -i Example/
```
*(This is typically called automatically by the Inference script, creating a `rules_by_gene.tsv` file in your output directory)*

### 3. Rule Combination Evaluator (`3.BNI3_Evaluate_rules.py`)
Because multiple candidate rules can tie for the top fitness score per gene, this script evaluates permutations of the best Boolean rules. It tests these combinations analytically (calculating basin entropies, cycle lengths, distances from Kauffman's optimal $K=2$, and final state concordance) to rank and determine the absolute best, functionally unified ruleset for your network.

**Basic Usage:**
```bash
python3 3.BNI3_Evaluate_rules.py -r Example/rules_by_gene.tsv -m ../1.Binarization/Example/bin_SSD.tsv
```
*This will output two files:*
*   `evaluation_results.tsv`: The full ranking and attractor metrics for all evaluated combinations.
*   `rules_by_gene_evaluated.tsv`: The definitive winning ruleset, ready to be passed to the Attractor stage.

### 4. Network Visualizer (`4.BNI3_Boolean_network_visualizer.py`)
Once you have the evaluated unified ruleset, this tool allows you to visually render the boolean network topology to formally inspect the interactions among regulators and genes.

**Basic Usage:**
```bash
python3 4.BNI3_Boolean_network_visualizer.py -i Example/rules_by_gene_evaluated.tsv -o Example/network_visualization.png
```
